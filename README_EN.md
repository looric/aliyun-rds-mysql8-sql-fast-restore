# Fast SQL Restore for Alibaba Cloud RDS MySQL 8.x Snapshot Backups

A zero-dependency Python tool for restoring SQL backup directories generated by
Alibaba Cloud RDS MySQL 8.x **Download Backup** into a self-managed MySQL 8.x
instance.

> Primary documentation is in [README.md](README.md). This English README is a
> concise quick-start guide for international users.

## Highlights

- **SQL-only**: designed for SQL backup directories, not CSV exports.
- **Zero dependencies**: Python 3.6.8+ standard library only.
- **Linux only**: tested on Alibaba Cloud Linux release 3 (OpenAnolis Edition).
- **Flat and nested layouts**: supports one-database and multi-database exports.
- **Table-level concurrency**: imports different tables in parallel while keeping
  each table's SQL parts ordered.
- **Generated column fix**: rewrites explicit values for generated columns to
  `DEFAULT` during import to avoid MySQL `ERROR 3105`.
- **Safe password handling**: does not support `MYSQL_PWD`; uses temporary
  `--defaults-extra-file`, `--password-file`, a user-provided option file, or
  `--login-path`.
- **Dry-run and fail-fast**: preview the restore plan and stop quickly on errors.

## Requirements

- Linux
- Python 3.6.8+
- MySQL 8.x server and MySQL 8.x client
- `zstd` and `tar` for extracting `.tar.zst` backup archives

## Download and extract an RDS backup

```bash
wget -b -c -O backup.tar.zst "<RDS_BACKUP_DOWNLOAD_URL>"
mkdir -p /home/mysql/data
zstd -d -c backup.tar.zst | tar -xvf - -C /home/mysql/data
```

Never commit real RDS backup URLs. They may contain temporary authorization data
or business-sensitive paths.

## Pre-checks

Enable `local_infile` on the target MySQL instance, as required by the official
Alibaba Cloud restore workflow:

```sql
SHOW GLOBAL VARIABLES LIKE 'local_infile';
SET GLOBAL local_infile=1;
```

For persistent configuration:

```ini
[mysqld]
local_infile=1
```

## Quick start

Preview the restore plan:

```bash
python3 restore_sql_fast.py /home/mysql/data 127.0.0.1 3306 root   --ask-password   --dry-run
```

Restore with table-level concurrency:

```bash
python3 restore_sql_fast.py /home/mysql/data 127.0.0.1 3306 root   --ask-password   --workers 4   --log-file restore.log
```

If the backup path contains spaces, non-ASCII characters, or shell-sensitive
symbols, use stream mode:

```bash
python3 restore_sql_fast.py "/home/mysql/rds backup" 127.0.0.1 3306 root   --ask-password   --input-mode stream
```

## Layouts

Single database, flat layout:

```text
/home/mysql/data/service2023db/
|-- structure.sql
|-- t_week_channel/
|   |-- structure.sql
|   `-- data/
|       `-- t_week_channel_0_part0.sql
`-- t_work_day/
    |-- structure.sql
    `-- data/
        `-- t_work_day_0_part0.sql
```

Run with the database directory:

```bash
python3 restore_sql_fast.py /home/mysql/data/service2023db 127.0.0.1 3306 root --ask-password
```

Multi-database, nested layout:

```text
/home/mysql/data/
|-- service2023db/
|   |-- structure.sql
|   `-- ...
|-- orderdb/
|   |-- structure.sql
|   `-- ...
`-- userdb/
    |-- structure.sql
    `-- ...
```

Run with the parent directory and do **not** pass `-D`:

```bash
python3 restore_sql_fast.py /home/mysql/data 127.0.0.1 3306 root --ask-password --workers 4
```

Restore only selected databases:

```bash
python3 restore_sql_fast.py /home/mysql/data 127.0.0.1 3306 root   --ask-password   --only-database service2023db,orderdb
```

## Recommended MySQL settings for a temporary restore instance

```ini
[mysqld]
local_infile=1
max_allowed_packet=1G
innodb_buffer_pool_size=70%-80%_RAM
innodb_redo_log_capacity=8G
innodb_log_buffer_size=256M
innodb_io_capacity=1000
innodb_io_capacity_max=2500
```

For a disposable restore instance, you may temporarily use:

```sql
SET GLOBAL innodb_flush_log_at_trx_commit = 2;
SET GLOBAL sync_binlog = 0;
```

Restore safer defaults after import:

```sql
SET GLOBAL innodb_flush_log_at_trx_commit = 1;
SET GLOBAL sync_binlog = 1;
```
